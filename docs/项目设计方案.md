# 项目设计方案

项目主题：AI 赋能的大学生课程小组协作辅助平台

## 1 项目愿景

针对大学生在课程小组协作中普遍存在的“任务设置与拆解模糊”、“进度失控”、“缺乏灵感与参考案例”等痛点，本平台通过 AI 深度赋能，致力于解决大学生课程协作中的双重核心诉求：

1. **管理愿景：**通过 **AI 辅助的项目树构建** 与 **动态依赖时间轴**，将复杂的学期任务拆解为可执行、有执行期限的最小单元，实现团队协作的透明化与高效化。
2. **灵感愿景：** 打造 **“全链条渗透”的灵感广场**，为任务的每个阶段（规划、执行）提供创意参考与成熟模板，降低协作的冷启动门槛，实现集体智慧的复用 。

## 2 核心场景与业务逻辑

### 2.1 项目树：递归式任务拆解逻辑

- 平台采用“树状结构”承载任务。AI 根据用户输入的原始要求，辅助将其从宏观目标拆解至原子操作。

- **示例：** 
  - **根任务：** 《习概》课程小组合作任务
  - **子任务（L1）：** 读书报告(6人) | 社会实践(6人) | 思想汇报(贯穿全程)
  - **孙任务（L2 - 读书报告）：** 文献阅读(2人) | PPT制作(2人) | 演讲汇报(2人)
  - **原子任务（L3 - 演讲汇报）：** 讲稿撰写(1人) | 上台演讲(1人) 

- 支持一名同学在不同任务节点中担任不同角色（如：读书报告中的“PPT制作” + 社会实践中的“领队” + 个人思想汇报可由同一位同学负责）

### 2.2 灵感广场：全链条渗透的资源库

灵感广场深度嵌入到任务推进的每一个节点：

- **启动期 (创生灵感)：** 在项目树还未配置时，组长可基于“灵感广场”的成熟模板进行微调，一键生成整个项目的结构 。
- **执行期 (辅助灵感)：** 每一个具体的原子任务（如“PPT制作”）都关联了灵感广场的相关条目。用户在查看任务详情时，可以一键跳转至对应的实现思路或参考案例 。
- **总结期 (沉淀灵感)：** 任务完成后，用户可将自己的优秀方案、学习经验等分享至灵感广场，完成从“消耗资源”到“创造资源”的闭环 。

### 2.3 成员身份与权限管理机制

- 一名成员可以加入多门不同的课程小组，在不同小组中担任不同的职务。
- 平台实行“组长负责制”以保障项目推进的有序性：
  - **组长权限：** 拥有项目树/时间轴的完全编辑权、任务详情编辑权、组员管理权（邀请/踢出/转让组长身份）、解散小组权。
  - **组员权限：** 仅拥有个人任务的状态流转权。

## 3 页面功能详细设计

设计如下四个页面：

### 3.1 用户首页（Dashboard）

首页是每位同学每天进入平台的首要入口，聚焦**个人执行视角**。

**主要功能：**

- **我的任务列表**：按 DDL时间或课程名排序，展示所有课程、所有任务的截止日期。
  
  - 点击任一任务项后，可在弹窗中查看：
    - 该任务在**项目树**中的位置
    - 该任务在**时间轴**中的位置
    - 任务具体描述、个人权责
    - 一键跳转至“灵感广场”，查看相关实现灵感
- **AI 智能提醒面板：**
  
  - **紧急预警：** 弹窗或横幅提示近期高优任务（例：“同学你好！你有一项紧急 DDL 将于明天截止，建议你...”）。
  
    **AI 个性化建议：** 根据分工和整体进度，生成今日建议（例：“建议今天完成文献阅读前3章”）。
  - **组长报告：**为组长额外提供**小组整体进度汇总报告**。

### 3.2 项目树页面（Project Tree & Timeline）

本页面是平台的核心管理工具，将**项目树**与**时间轴**融合在同一页面内，实现结构化拆解与可视化跟踪的统一。

#### 3.2.1页面顶部

- 课程切换条（支持快速在不同课程的项目树间切换）
- 视图切换器：在项目树视图与时间轴视图中进行切换

#### 3.2.2 项目树视图

- **层级展示：** 以树状目录结构直观展示任务层级（如：习概 -> 实践调研 -> PPT制作）。
- **专注模式：** 提供 **“仅看与我有关的模块”** 筛选开关，帮助组员在庞大的项目树中迅速聚焦。

#### 3.2.3 时间轴视图

- **甘特图排期表：**
  - **纵轴（人员）：** 按组内成员（如：小明、小红）进行泳道划分。
  - **横轴（时间）：** 支持 **周视图 / 月视图** 切换。

- **任务块映射：** 将项目树底层的“原子任务”（如“PPT制作”、“现场调研”）作为彩色区块，精准映射至对应人员的日期区间内。

#### 3.2.4 项目树的构建与编辑

- **初始创建：**
  - 由组长进行。
  - 方法一：从**灵感广场**一键导入相似课程的成熟项目树模板，并通过AI对话进行微调
  - 方法二：上传课程要求文档（PDF/Word/图片），并通过AI对话自动生成初始项目树
  - 关于分工：可以根据组员设置的技能标签，由AI自动为组员分配任务。也可随机分配任务。
- **后续编辑：**
  - 支持增删改节点，由组长进行。
  - 方法一：在视图中点击节点，直接唤起任务详情弹窗进行编辑
  - 方法二：点击页面的“编辑”按钮，通过与 AI 对话的方式完成编辑（如：“把演讲汇报再拆成上台演讲和传单讲解两部分”）。
  - 方法三：支持删除整颗项目树，即“推倒重来”。
  
- **迭代策略**：
  - 先建立粗粒度的大纲（大枝干），后续逐步细化子任务，符合实际项目演进过程。
  - 每个节点会有一个属性，用于记录其是否已被细化为原子任务。

- 项目树是基础，时间轴是由细化后的项目树生成的。仅组长可调节。

### 3.3 灵感广场页面（Inspiration Plaza）

灵感广场是平台的**创意与知识共享社区**，致力于降低小组作业的启动与推进成本。

**主要内容类型：**

- 课程协作模板（完整项目树模板、分工方案、时间规划）
- 优秀案例分享（读书报告框架、社会实践方案、PPT模板、演讲脚本等）
- 课程特定干货（文献推荐、常见坑点避雷、老师偏好分析等）
- 任意与课程相关的灵感、经验、工具推荐

**主要功能模块：**

- **全局搜索与精准过滤：**
  - 顶部提供全局搜索框（“搜索课程和资源”）。
  - 支持多维度筛选：**按课程浏览**、**按类别浏览**、**最新发布**。

- 支持上传/编辑/删除资料，支持点赞、收藏、评论等社交互动功能，鼓励学生将每次的成功经验转化为可复用的集体智慧。

### 3.4 我的页面（My Profile）

#### 3.4.1 Tab A: 我的信息

- **基础档案：** 头像、姓名、学号、专业。
- **AI 技能雷达：**
  - 用户可自定义或由平台自动打上技能标签（如：PPT制作、绘画、摄影、统筹）。
  - **业务联动：**在项目树初始化阶段，AI 将读取此标签为组长提供智能任务分配建议。
- **账户安全设置：** 修改密码、修改手机号、退出登录。

**3.4.2 Tab B: 我的小组**

- **小组列表视图：** 按课程维度展示用户加入的所有小组，并在列表前端显性标注当前角色（**组长** 或 **组员**）。
- **新建入口：** 提供全局“新建小组”快捷按钮。
- **小组管理菜单（点击单列展开）：**
  - 邀请组员（*仅组长可见*）
  - 组员管理（*仅组长可见*）
  - 组长转让（*仅组长可见*）
  - 退出小组
  - 解散小组（*仅组长可见*）
